Implementaremos una serie de técnicas para mejorar la seguridad de nuestro código evitando que se filtren secretos mediante Gitleaks (https://github.com/gitleaks/gitleaks-action).

  • Automatizaremos mediante un Git Action su funcionamiento.
  • Lo ejecutaremos antes de hacer un commit en local.
  • Lo configuraremos (si es necesario) para evitar falsos positivos y reconocer secretos.

Documenta su uso en tu repositorio, mostrando como funciona.

1. Qué es Gitleaks

Gitleaks es una herramienta multiplataforma de Análisis de Código Estático (SAST) de código abierto, diseñada específicamente para detectar contraseñas pasadas, claves API y tokens en repositorios Git.

Analiza el repositorio local o remoto mediante expresiones regulares (regex) predefinidas o personalizadas y entropía de cadenas para identificar patrones que parezcan secretos subidos por error.

Puede ejecutarse localmente como un hook de pre-commit (bloqueando la creación del commit en la propia máquina del desarrollador si detecta un secreto) o integrarse en el pipeline de CI/CD para rechazar un push que contenga información sensible.

1.1. ¿Qué hacer si Gitleaks detecta algo?

Si la herramienta lanza una alerta, el flujo de trabajo debe ser:

Invalidar el secreto
Cambiar la contraseña o revocar el API Key inmediatamente. Borrar el commit no es suficiente, el secreto ya debe considerarse comprometido.
Limpiar el historial
Si es estrictamente necesario borrar el rastro, se deben usar herramientas como git filter-repo o el BFG Repo-Cleaner.

1.2. Beneficios de usar Gitleaks en el equipo

Evita multas y filtraciones
Previene el acceso no autorizado a infraestructura cloud (como AWS o Azure).
Educación
Los desarrolladores aprenden a usar variables de entorno o gestores de secretos (como GitHub Secrets) en lugar de hardcodear credenciales.
Auditoría
Deja un registro de que el equipo sigue estándares de seguridad en el ciclo de vida de desarrollo.

2. Uso

2.1. Automatización con GitHub Actions

La forma más robusta de asegurar que ningún secreto llegue a la rama principal (main). Puedes usar la acción oficial de Gitleaks. 

name: gitleaks
on:
  pull_request:
  push:
    branches: [main]
jobs:
  scan:
    name: gitleaks
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0 # Importante para escanear todo el historial
      - uses: gitleaks/gitleaks-action@v2
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

Haz una prueba poniendo un secreto en el repositorio y documenta como la Action ha funcionado.

2.2. Uso Local (Pre-Commit)

Lo ideal es detectar el secreto antes de que salga de la computadora del desarrollador.

2.2.1. Instalación

Leer:

2.2.2. Escaneo manual

Ejecuta este comando en la raíz de tu proyecto para revisar todo el historial: 

gitleaks detect --verbose

Incorpora esta funcionalidad a tu proyecto de forma local para no "ensuciar" el historial. Documenta su funcionamiento.

2.3. Falsos positivos

A veces, Gitleaks puede dar "falsos positivos" (detectar algo que parece un secreto pero no lo es). Para gestionar esto, crea un archivo .gitleaks.toml en la raíz:

Allowlist
Puedes definir huellas digitales específicas o archivos que deben ser ignorados.
Reglas propias
Si tu empresa usa un formato de token específicas (ej. CORPID12345), puedes añadir una expresión regular para que Gitleaks lo busque específicamente.
Añadir al Allowlist
Solo si estás 100% seguro de que es un falso positivo.
# This is the default gitleaks configuration file.
# Rules and allowlists are defined within this file.
# Rules instruct gitleaks on what should be considered a secret.
# Allowlists instruct gitleaks on what is allowed, i.e. not a secret.
title = "gitleaks config"

[allowlist]
description = "global allow lists"
regexes = [
  '''219-09-9999''',
  '''078-05-1120''',
  '''(9[0-9]{2}|666)-\d{2}-\d{4}''',
]
paths = [
  '''gitleaks.toml''',
  '''(.*?)(jpg|gif|doc|pdf|bin|svg|socket)$''',
  '''(go.mod|go.sum)$''',
  '''vendor''',
]
[[rules]]
description = "Discord API key"
id = "discord-api-token"
regex = '''(?i)(?:discord)(?:[0-9a-z\-_\t .]{0,20})(?:[\s|']|[\s|"]){0,3}(?:=|>|:=|\|\|:|<=|=>|:)(?:'|\"|\s|=|\x60){0,5}([a-f0-9]{64})(?:['|\"|\n|\r|\s|\x60]|$)'''
secretGroup = 1
keywords = [
  "discord",
]

Configura si es necesario un .gitleaks.toml para tu proyecto evitando falsos positivos o instruyendo al programa sobre cómo reconocer un secreto.
