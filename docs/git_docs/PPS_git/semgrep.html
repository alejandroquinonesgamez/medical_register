1. Introducción

Esta actividad tiene como objetivo comprender qué es Semgrep, por qué es una herramienta esencial en el ecosistema de DevSecOps y cómo implementarla en un flujo de trabajo real.

2. ¿Qué es Semgrep?

Semgrep (Semantic Grep) es una herramienta de análisis estático de código (SAST) de código abierto. A diferencia de un grep tradicional que busca cadenas de texto planas, Semgrep entiende la estructura sintáctica del código (el Árbol de Sintaxis Abstracta o AST).

Esta herramienta tiene tanta funcionalidad que se solapa con otras. Por ejemplo, Semgrep puede buscar secretos o buscar problemas en la cadena de suministro (y por supuesto tiene IA).

2.1. Utilidad y Ventajas

Detección de vulnerabilidades
Encuentra patrones inseguros (como inyecciones SQL o uso de funciones obsoletas).
Cumplimiento de estándares
Asegura que el equipo siga las mejores prácticas de codificación.
Personalización
Permite escribir reglas propias en segundos usando una sintaxis que se parece al código real.
Velocidad
Es extremadamente rápido comparado con herramientas SAST tradicionales.

4. Escaneos gestionados (managed scans)

Los análisis gestionados de Semgrep permiten ejecutar análisis de Semgrep sin necesidad de configurar ni mantener una infraestructura propia. Proporcionan una forma sencilla y escalable de analizar el código en busca de vulnerabilidades de seguridad, problemas de calidad y otros inconvenientes, sin tener que configurar ni mantener configuraciones independientes para cada proyecto.

Nuestro código tiene que estar los repositorios de:

  • Bitbucket
  • Github
  • Gitlab
  • Azure

Para configurar los escaneos necesitamos:

  • Cuenta en la organización (por ejemplo Github)
  • Seguir las instrucciones para escaneos gestionados

5. Ejemplos de utilización

Semgrep se puede utilizar de tres formas principales:

5.1. Escaneo Ad-hoc

Ideal para una revisión rápida de un directorio. 

semgrep scan --config auto

El flag --config auto busca reglas recomendadas por la comunidad para los lenguajes detectados en tu carpeta.

5.2. Uso de Reglas Específicas

Si solo quieres buscar vulnerabilidades web en un proyecto Python: 

semgrep scan --config "p/python" --config "p/owasp-top-10"

5.3. Creación de Reglas Personalizadas

Imagina que quieres prohibir el uso de print() en producción. Puedes crear un archivo my-rule.yaml: 

rules:
  - id: no-print-in-prod
    patterns:
      - pattern: print(...)
    message: "No uses print() en el código final. Usa un logger."
    languages: [python]
    severity: WARNING

5.4. Integración en CI/CD

Seguiremos las instrucciones de añadir semgrep al CI. Esto suele ser añadir al pipeline de GitHub Actions un fichero .github/workflows/semgrep.yml: 

# Name of this GitHub Actions workflow.
name: Semgrep

on:
  # Scan changed files in PRs (diff-aware scanning):
  pull_request: {}
  # Scan on-demand through GitHub Actions interface:
  workflow_dispatch: {}
  # Scan mainline branches if there are changes to .github/workflows/semgrep.yml:
  push:
    branches:
      - main
      - master
    paths:
      - .github/workflows/semgrep.yml
  # Schedule the CI job (this method uses cron syntax):
  schedule:
    - cron: '20 17 * * *' # Sets Semgrep to scan every day at 17:20 UTC.
    # It is recommended to change the schedule to a random time.

permissions:
  contents: read

jobs:
  semgrep:
    # User definable name of this GitHub Actions job.
    name: semgrep/ci
    # If you are self-hosting, change the following `runs-on` value:
    runs-on: ubuntu-latest

    container:
      # A Docker image with Semgrep installed. Do not change this.
      image: semgrep/semgrep

    # Skip any PR created by dependabot to avoid permission issues:
    if: (github.actor != 'dependabot[bot]')

    steps:
      # Fetch project source with GitHub Actions Checkout. Use either v3 or v4.
      - uses: actions/checkout@v4
      # Run the "semgrep ci" command on the command line of the docker image.
      - run: semgrep ci
        env:
          # Connect to Semgrep AppSec Platform through your SEMGREP_APP_TOKEN.
          # Generate a token from Semgrep AppSec Platform > Settings
          # and add it to your GitHub secrets.
          SEMGREP_APP_TOKEN: ${{ secrets.SEMGREP_APP_TOKEN }}

Fíjate que tendrás que crear un token e introducirlo en los secretos del repositorio. Puede que la versión gratuita no tenga esta funcionalidad activada (paga $$).
