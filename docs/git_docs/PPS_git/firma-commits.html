En esta actividad aprenderas a firmar tus commits con una clave GPG.

Documenta la actividad mostrando al final como tus commits aparecen firmados en el repositorio.

1. Introducción

La firma de commits añade una capa de no repudio y autenticación al control de versiones utilizando criptografía asimétrica (como GPG, SSH o S/MIME). Asegura que el autor del commit es realmente quien dice ser y que el contenido no ha sido alterado después de la firma. Sin esto, cualquier persona podría configurar su `user.name` y `user.email` como "Elon Musk" y Git lo aceptaría sin rechistar.

Identidad verificada
Es trivial configurar un cliente de Git local para firmar con un correo electrónico y nombre arbitrario para intentar hacer pasar un código malicioso como si lo hubiera aportado otra persona (suplantación de identidad). Al firmar criptográficamente el commit con tu clave privada, garantizas matemáticamente que eres tú realmente el autor.
Integridad
La firma asegura también que el contenido del commit no ha sido alterado desde que fue transmitido. Plataformas como GitHub muestran visualmente un distintivo verde "Verified" en los commits que tienen una firma válida, aumentando en gran medida la confianza en la línea de código base.

2. Generar una llave GPG

El estándar más común para firmar en Git es GPG (GNU Privacy Guard).

2.1. Instalar GPG:

En macOS: 

brew install gpg2

En Windows:

Desinstala Windows e instala algo mejor o descarga GPG4Win.

Linux (suele venir instalado): 

sudo apt install gnupg

2.2. Generar el par de llaves

gpg --full-generate-key

Elige la opción por defecto, un tamaño de 4096 bits y que no caduque (o ponle una fecha de expiración).

3. Obtener el ID de tu llave

gpg --list-secret-keys --keyid-format=LONG

/home/fraya/.gnupg/pubring.kbx
------------------------------
sec   ed25519/CF07E6A6005C1E55 2026-04-16 [SC] [caduca: 2035-04-14]
      CDD2FF18C1C3E8194CC711F0CF06E6A6005C1E55
uid              [  absoluta ] Fernando Raya (IES Zaidín-Vergeles) <fraya@ieszaidinvergeles.org>
ssb   cv25519/756902B776EC435E 2026-04-16 [E] [caduca: 2035-04-14]

El ID es CF07E6A6005C1E55.

4. Configurar Git para usar tu llave

Ahora debes decirle a Git qué llave debe usar para firmar tus commits. 

# Configura el ID de tu llave
git config --global user.signingkey CF07E6A6005C1E55

# (Opcional) Configura Git para que firme TODOS los commits automáticamente
git config --global commit.gpgsign true

5. Añadir la llave pública a GitHub

Para que GitHub muestre el badge de "Verified", necesita conocer tu llave pública.

5.1. Exportar la llave:

gpg --armor --export 3AA5C34371567BD2

Copia todo el bloque de texto que empieza con 

-----BEGIN PGP PUBLIC KEY BLOCK-----

5.2. Pegarla en GitHub:

Ve a Settings > SSH and GPG keys > New GPG key. Pega el bloque de texto y guarda.

6. Firmar commits

Si no activaste la firma automática en el paso 2, debes añadir el flag -S al hacer el commit: 

git commit -S -m "Mi mensaje de commit seguro"

Git te pedirá la contraseña que configuraste al generar la llave GPG.

7. El modo "Vigilante" (Vigilant Mode)

GitHub tiene una opción avanzada en la configuración de seguridad llamada Vigilant Mode.

Sin Vigilant Mode
GitHub solo pone el badge "Verified" a los commits firmados. Los no firmados aparecen normales.
Con Vigilant Mode
GitHub marca los commits firmados como "Verified" pero los que no están firmados (o están mal firmados) aparecen con un badge de "Unverified".

Esto es extremadamente útil para equipos de alto perfil, ya que hace que cualquier commit sospechoso resalte inmediatamente.
